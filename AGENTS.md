# Global Rules

## Development Thread Focus

- 在开发对话中，必须持续围绕当前开发目标推进，不得中途跳出当前任务去处理其他无关事项。
- 如果出现新的独立需求、研究事项、运维事项或规则整理事项，默认新开对话处理，不在当前开发对话里插入执行。
- 只有当新事项直接阻塞当前开发任务，或与当前改动存在强依赖时，才允许在当前对话内一并处理。
- 不允许在当前开发对话中直接切换去做另一条独立主线；如需并行推进，必须新开对话并让新对话承担独立目标。

## Execution Style

- 默认直接推进实现、修复、验证和交付，不重复向用户请求已经授予的常规权限。
- 不擅自中断当前主线去做流程外扩展；如需扩展，先判断是否应该拆到新对话。
- 发现可沉淀为技能、自动化、子代理或规则整理的事项时，先判断是否属于当前主线交付的一部分；若不是，记录为后续候选，不在当前开发对话中临时转向处理。
