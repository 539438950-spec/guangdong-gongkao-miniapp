# 首版数据源

- 广东省人事考试网：`https://rsks.gd.gov.cn/`
- 广东省人社公共服务入口：`https://ggfw.hrss.gd.gov.cn/`
- 国考专题入口：`https://bm.scs.gov.cn/`

## 当前接入状态

- `rsks-gd`
  - 状态：真实官方抓取已接入。
  - 当前前台能力：可展示公告，可开放岗位列表、筛选、岗位对比和规则推荐。
  - 当前实现依据：岗位表已完成结构化，现有快照里可解析出岗位数据并进入前台读库。
- `ggfw-hrss-gd`
  - 状态：真实官方公告抓取已接入，但当前仍是 `attachment-only`。
  - 当前前台能力：只展示公告和附件，不开放岗位列表、岗位对比和推荐。
  - 当前上线边界：在没有稳定结构化岗位表之前，必须保持公告模式，不能带病开放选岗能力。
- `national-bm`
  - 状态：当前仍为 `demo` 源，不是正式官方接入。
  - 当前前台能力：仅用于产品演示，前台必须显式标记“演示”。
  - 当前上线边界：由于当前运行环境无法稳定直连 `bm.scs.gov.cn`，不得当作正式生产来源对外宣称已接入。

## 当前结论

首版当前真实可上线的边界应按下面执行：

- `rsks-gd`：公告 + 岗位表都可上线。
- `ggfw-hrss-gd`：仅公告可上线。
- `national-bm`：仅演示，不计入正式来源覆盖。

## 来源接入原则

- 只有公告和岗位表都能稳定结构化时才开放岗位对比。
- 没有结构化成功的岗位表时，只能保留公告模式，不能开放岗位能力。
- 每个来源独立适配器，单独监控失败率。
- 解析失败时保留原始页面和附件索引，进入人工复核队列。
- 演示源必须与正式来源显式区分，不能混淆为生产接入状态。
